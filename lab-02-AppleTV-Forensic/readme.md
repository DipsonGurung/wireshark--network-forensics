## Overview

This lab analyses a packet capture taken from a covert surveillance operation targeting Ann's AppleTV device (`192.168.1.10`). Ann and Mr. X had established a new base of operations while extradition paperwork was being processed. All network traffic generated by the AppleTV was captured and analysed to reconstruct browsing behaviour, search activity, and media interests.

The AppleTV (firmware 2.4) communicates with Apple's iTunes Store over **unencrypted HTTP**, meaning every search query, movie selection, and server response is fully readable in the capture — no decryption required.

## Forensic Questions & Answers

### Q1 — MAC Address of Ann's AppleTV
**Answer:** `00:25:00:fe:07:c4`  
Found in ARP reply at Packet 3. The OUI prefix `00:25:00` is registered to Apple Inc. The device also announces itself as `AppleTV.local` via mDNS (Packet 36).

**Wireshark filter:** `arp`

---

### Q2 — User-Agent String
**Answer:** `AppleTV/2.4`  
Sent with every HTTP request from `192.168.1.10`. Identifies the device as a **2nd-generation Apple TV** running software version 2.4.

**Wireshark filter:** `frame.number == 6`

---

### Q3 — First Four Search Terms (Incremental)
The AppleTV sends a new HTTP GET request to `ax.search.itunes.apple.com` after **every single keystroke**. This exposes each character typed to the network.

| # | Search Term | Packet | URI |
|---|---|---|---|
| 1 | `h` | 43 | `incrementalSearch?media=movie&q=h` |
| 2 | `ha` | 180 | `incrementalSearch?media=movie&q=ha` |
| 3 | `hac` | 230 | `incrementalSearch?media=movie&q=hac` |
| 4 | `hack` | 276 | `incrementalSearch?media=movie&q=hack` |

**Answer:** `h → ha → hac → hack`

**Wireshark filter:** `http.request.uri contains "incrementalSearch" && frame.number <= 280`

---

### Q4 — First Movie Clicked
**Answer:** **Hackers (1995, dir. Iain Softley)**  
At Packet 307, the AppleTV fetches movie ID `333441649`. The XML response at Packet 312 confirms the title. Metrics ping at Packet 320 also confirms: `pageName=Movie%20Page-US-Hackers-Iain%20Softley-333441649`

**Wireshark filter:** `frame.number == 307`

---

### Q5 — Movie Trailer URL
**Answer:**  
`http://a227.v.phobos.apple.com/us/r1000/008/Video/62/bd/1b/mzm.plqacyqb..640x278.h264lc.d2.p.m4v`  
H.264 widescreen trailer at 640×278 resolution, hosted on Apple's Phobos CDN servers.

**Wireshark filter:** `frame.number == 312` → expand XML → find `preview-url`

---

### Q6 — Second Movie Clicked
**Answer:** **Sneakers (1992, dir. Phil Alden Robinson)**  
Ann typed `s → sn → sne → sneb → snea → sneak`. At Packet 1181, the AppleTV fetches movie ID `283963264`. Confirmed by metrics URL at Packet 1189: `pageName=Movie%20Page-US-Sneakers-Phil%20Alden%20Robinson-283963264`

**Wireshark filter:** `frame.number == 1181`

---

### Q7 — Price to Buy Sneakers
**Answer:** `$9.99` (Standard Definition buy price)  
Found in the `price-display` field of the iTunes Store XML response at Packet 1186. Rental SD was also available at $2.99.

**Wireshark filter:** `frame.number == 1186`

---

### Q8 — Last Search Term
**Answer:** `iknowyourewatchingme`  
Decoded: *"I know you're watching me"* — Ann typed this phrase character by character across 21 incremental search requests, ending at Packet 1766. This is a significant intelligence indicator suggesting counter-surveillance awareness.

**Wireshark filter:** `frame.number == 1766`

## Key Findings

- HTTP traffic from the AppleTV is **entirely unencrypted** — every search query, movie selection, and server response is visible without decryption.
- **Incremental/keystroke search** requests leak the user's full intended term even if they never pressed "Search" — each character is transmitted individually.
- iTunes Store XML responses expose rich metadata: movie titles, prices, ratings, trailer URLs, and purchase parameters — all recoverable from a single packet.
- The device's **MAC address** (OUI: Apple Inc.) and mDNS announcements allow positive identification of the device model without any login credentials.
- Ann's final search `iknowyourewatchingme` is a significant intelligence indicator and should be escalated.
- No HTTPS/TLS was used — consistent with older AppleTV 2.4 firmware. Newer Apple devices encrypt all store traffic.
